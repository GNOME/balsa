Balsa download

Requirements

  1. A basic GNOME installation.
  2. Since version 1.1.5, we use libESMTP library for message submission. It is small (<100kB) but modern and resourceful.
  3. Balsa 0.9.5 and greater uses the libpspell library for spell checking, which is available in both rpm and deb format as well as plain source. You can find the library and modules at the pspell website.
  4. Make sure you have libtool package installed. It usually comes with your distribution.
  5. You may want to recompile balsa to get: HTML support (--enable-gtkhtml), GSS support (--with-gss) or LDAP support (--enable-ldap).

Download

balsa-1.0.0 released on 2000-11-11 is the most recent stable release. It has passed long testing period and should be quite solid. Anyway, a lot has changed since then and if you look for reliable IMAP access and other goodies, try balsa-1.1.4 or later instead.

The binary rpm 1.1.7 is compiled on RH-7.1 with update packages installed, and with gnome-print-0.25 (available from Ximian). You can get rid of this dependency by recompiling balsa. You may want to recompile balsa (you may find rpm --rebuild balsa*src.rpm or rpm -ta balsa.tar.bz2 handy) to enable LDAP, GSS or HTML support, which are disabled by default.

There have been many improvements over the last stable release, including:

  • Improved message viewer
  • Updated backend for handling mailbox operations, now much more robust and faster.
  • Multiple address book support, both local vCard and remote LDAP books are supported
  • Alias-expansion
  • CRAM and GSSAPI Kerberos authentication support
  • Spell checking
  • New charset support
  • Nifty quoted text colouration
  • Menu reorganization
  • Better threading (the multiple process-kind)
  • Use gnome-print for printing
  • Use gnome-config now instead of libProplist
  • A metric shwack of bug fixes
  • Y2K compliant :-)

CVS source & build instructions

Balsa source is also available from GNOME CVS repository. Check out balsa module. The CVS version is sometimes not sufficiently stable for everyday use it but has all the new features and is best starting point if you want to hack balsa.

You will need pspell-devel (obligatory) and gnome-print-devel (optional) packages for building balsa. Get also gnome-doc-tools or alternatively install png-support-3.?.dtd and gdp-both.dsl files yourself. Make sure your have libtool installed.

If you get errors compiling balsa help file balsa.sgml, make sure your SGML catalog file contains paths to relevant packages. The frequent error is to omit PNG support files which results in seemingly unrelated errors.

Balsa download

Requirements

  1. A basic GNOME installation.
  2. Balsa needs libPropList. Check if you don't have it already on your system, subsequently try to download suitable rpm. If you cannot do it for some reason, grab libPropList.tar.gz, It must be installed with a prefix of /usr if you wish to also use a Balsa RPM. Note, that in this last case you might need to ignore dependices warnings when installing rpm.
  3. balsa-0.9.4 binary rpms are compiled on RH-6.2 with update packages installed. The binaries come with GSS authorisation enabled. You need Kerberos libraries (krb5-libs and krb5-config) provided with RH-6.2 and up, for example. Alternatively, you can recompile balsa from the source (man rpm is your friend) with GSS disabled.

Experimental release - 0.9.4

2000.09.05: This is pretty much all we have in CVS: new message browser, improved address book handling, more reliable mailbox backend, alias expansion. The code has however some experimental and unfinished features enabled so be warned! The POP problems and message deletion crashes have been finally addressed. We have also now GSS and CRAM-MD5 authorization for IMAP, and APOP for POP3.

CVS source

Balsa source is also available from GNOME CVS repository. Check out balsa module. The CVS version is sometimes not sufficiently stable for everyday use it but has all the new features and is best starting point if you want to hack balsa.

Balsa download

Requirements

  1. A basic GNOME installation.
  2. Since version 1.1.5, we use libESMTP library for message submission. It is small (<100kB) but modern and resourceful. We have precompiled binaries (compiled on stock RH-7.1) or source rpm packages (both for libesmtp-0.8.4) ready for you.
  3. Balsa 0.9.5 and greater uses the libpspell library for spell checking, which is available in both rpm and deb format as well as plain source. You can find the library and modules at the pspell website.
  4. Make sure you have libtool package installed. It usually comes with your distribution.
  5. You may want to recompile balsa to get: HTML support (--enable-gtkhtml), GSS support (--with-gss) or LDAP support (--enable-ldap). You will need corresponding development modules.

Download

The binary rpm is compiled on RH-7.1 with update packages installed. You may want to recompile balsa (you may find rpm --rebuild balsa*src.rpm or rpm -ta balsa.tar.bz2 handy) to enable LDAP, GSS or HTML support, which are disabled by default.

CVS source & build instructions

Balsa source is also available from GNOME CVS repository. Check out balsa module. The CVS version is sometimes not sufficiently stable for everyday use it but has all the new features and is best starting point if you want to hack balsa.

You will need pspell-devel (obligatory) and gnome-print-devel (optional) packages for building balsa. Get also gnome-doc-tools or alternatively install png-support-3.?.dtd and gdp-both.dsl files yourself. Most likely, it will be necessary to add to your /etc/sgml/docbook.cat (or whatever catalog file your db2html tool uses) a pointer to /usr/share/sgml/docbook/png-support-3.1/catalog! Also, make sure your have libtool installed.

If you get errors compiling balsa help file balsa.sgml, make sure your SGML catalog file contains paths to relevant packages. The frequent error is to omit PNG support files which results in seemingly unrelated errors.

If you want to compile from the CVS source, you will need gob.
